Pseudo-code — QuickSort

quicksort(arr, key, lo, hi):
  if lo < hi:
    pivot_index = partition(arr, key, lo, hi)
    quicksort(arr, key, lo, pivot_index - 1)
    quicksort(arr, key, pivot_index + 1, hi)

partition(arr, key, lo, hi):
  pivot = arr[hi][key]
  i = lo - 1
  for j in range(lo, hi):
    if arr[j][key] >= pivot:   ← descending order
      i++
      swap(arr[i], arr[j])
  swap(arr[i+1], arr[hi])
  return i + 1

Rejection Reasons

Data Cleaning Decisions

  • Negative fares — Meter errors or refund records. Excluded from analysis.
  • Zero passengers — Data entry errors. Standard is minimum 1 passenger.
  • Zero distance — Cancelled trips or GPS failures. No meaningful analysis possible.
  • Future dates — Timestamps beyond January 2024 are temporal anomalies.
  • Speed > 100 mph — Physically impossible for NYC taxi. GPS errors.
  • Null passengers/ratecode — Filled with mode (1) rather than excluded.
